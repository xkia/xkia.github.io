PostgreSQL 的dump操作 ​

常用操作 ​

bash
# 基本操作
pg_dump -h <host> -U <username> -s -t <table_name> <database_name> > <output_file_path>

# 仅导出表结构
pg_dump -U username -h host -p port -s --no-owner --no-comments --no-set -t table1,table2 -f output.sql dbname

# 仅导出表数据
pg_dump -U username -h host -p port -a --no-owner --no-comments --no-set -t table1,table2 -f output.sql dbname

解释： ​

  • -U username：指定数据库用户名。
  • -h host：指定数据库主机。
  • -p port：指定数据库端口。
  • -s：仅导出数据库结构，不包括数据。
  • --no-owner：不导出对象所有者信息。
  • --no-comments：不导出注释。
  • --no-set：不导出默认的 SET 设置。
  • -t table1 -t table2：仅导出 table1 和 table2 的结构；不使用 -t 参数时，导出整个数据库的结构。
  • -f output.sql：指定导出文件路径。
  • dbname：指定要导出的数据库名。
  • -a 或 --data-only：仅导出数据，不导出表结构。
